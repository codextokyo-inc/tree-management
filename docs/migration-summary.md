## ⚠️ 残っている作業（軽微な調整）

### 1. データ入出力の調整
現在、以下の関数がまだ旧フォーマットを参照している可能性があります：
- CSV エクスポート
- CSV インポート
- JSON 保存/読込

**影響**: 旧形式のデータを読み込むとエラーになる可能性

**対処**: 
- エクスポート時に樹木用フィールドのみ出力
- インポート時に不要フィールドを無視
- データ正規化処理の追加

### 2. ポリゴン集計機能
現在の集計機能はビジネス用（設備金額、従業員数等）を集計します。

**対応方針**:
- 樹木のカウント
- 樹種別の内訳
- エリア内の樹木一覧表示

### 3. 住所検索の色設定
`typeColors` マッピングがまだ旧タイプを参照しています。

```javascript
// 現在（旧）
const typeColors = {
    'hq': '#ff0000',
    'branch': '#ff4444',
    ...
};

// 必要な変更
const typeColors = {
    'tree': '#22bb33',
    'waypoint': '#3388ff'
};
```

### 4. フォームデータ処理
以下の関数を調整する必要があります：
- `loadAssetToForm()`: フォームにデータを読み込む
- `saveAssetFromForm()`: フォームからデータを保存
- `updateEmployeeCountLabel()`: 不要な関数の削除

## 📝 メモ: 今回の移植で学んだこと

### うまくいった点
- ベースコードのコピーから始めることで、動作するものを早期に作成できた
- 段階的な変更により、リスクを抑えながら進められた
- 明確な要件（樹種、登録日のみ）により、シンプルな実装になった

### 改善の余地
- 10,000行の単一HTMLファイルは保守が大変
- 機能ごとの独立性が低く、影響範囲の特定が困難
- 不要な機能が残っており、コードが肥大化

### Phase 2 への示唆
- ファイル分割は必須
- 機能ごとのモジュール化が重要
- config ベースの設計で差異を吸収
